# MVP_SCOPE（2026-04-08）

## MVP 目标

做一个本地可运行的简历检查与分流工作台，先保证主流程跑通：

`首页 → Diagnose → 模块执行 → 输出结果 + 下一步建议`

这批知识库是简历检查 MVP 的评审规则与素材来源，同时为后续 AIPM 全流程文档检查铺垫。

## P0（必须完成）

### 1. 主流程打通
- 首页入口统一先进入 `/diagnose`
- Diagnose 负责任务识别与路由
- 分流到模块页面后可继续执行

### 2. Resume 模块可独立闭环
- 文件上传（仅 PDF/DOCX）
- 自动解析文本
- 评审结果输出：
  - 总分（100）
  - 分角色评分（HR / PM / 技术 / 设计 / 项目管理）
  - 问题分级（P0 / P1 / P2）
- 改写结果输出：
  - 可直接替换文案
  - 完整优化稿

### 3. 下一步建议
- 在评审结果页、改写结果页给出下一步入口
- 可跳转到项目封装或面试准备链路

## P1（已完成）

### 1. 改写结果升级
- 增加双视图：
  - 分析视图
  - 简历成稿视图
- 增加投递清单（delivery checklist）

### 2. 其他模块骨架
- `/project`、`/interview`、`/workflow`、`/planner` 页面可访问
- 对应 API 已可返回结构化结果（MVP 级）

## 当前明确不做（Out of Scope）

- 登录系统
- 数据库
- 云部署
- 用户管理
- 多租户/协作权限

## 关键验收标准

1. 首页是否全部先经过 Diagnose 再进入模块
2. Resume 上传-评审-改写链路是否稳定可用
3. 结果页是否有明确下一步引导
4. 在不引入数据库/登录的前提下，本地可重复运行

## 本地手工测试建议

1. 访问首页，点击任一入口，确认先到 `/diagnose`
2. Diagnose 填写问题后，确认被分流到目标模块
3. 在 `/resume` 上传 PDF 或 DOCX，确认跳转 `/review`
4. 从 `/review` 进入 `/rewrite`，确认双视图与投递清单可见
5. 从结果页点击下一步建议，确认能跳转 `/project` 或 `/interview`
